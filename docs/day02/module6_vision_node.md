# 모듈 6: 실시간 카메라 비전 인식 연동 가이드

이 문서는 모듈 5에서 스폰한 Turtlebot3 로봇의 **카메라 실시간 영상(`/camera/rgb/image_raw`)**을 파이썬 코드에서 구독(Subscribe)하여, 로봇의 시야에 QR 코드가 들어오는 순간 데이터를 낚아채어 파싱하는 테스트 과정을 다룹니다.

---

## 1. 비전 인식 스크립트 권한 부여

새롭게 작성된 `vision_recognizer_node.py` 파일을 리눅스 가상머신으로 동기화(복사)한 뒤 실행 권한을 줍니다. 이 노드는 `cv_bridge` 패키지를 활용해 ROS 영상 포맷을 OpenCV가 다룰 수 있는 형태로 초당 수십 번씩 변환합니다.

```bash
# 실행 권한 부여
chmod +x ~/catkin_ws/src/qr_logistics_robot/scripts/vision_recognizer_node.py
```

---

## 2. 시뮬레이션 및 로봇 조종 환경 세팅

실제로 로봇이 맵 안을 움직이며 카메라로 QR을 찾아야 하므로, 여러 개의 터미널 창을 띄워놓고 작업해야 합니다.

### [터미널 1] 시뮬레이션 (Gazebo) 유지
방금 모듈 5에서 테스트하셨던 Gazebo 창이 켜져있다면 그대로 둡니다. (꺼져있다면 `roslaunch qr_logistics_robot logistics_world.launch` 로 켭니다.)

### [터미널 2] RViz (로봇 시야 확인창) 실행
로봇이 지금 무엇을 보고 있는지 확인하기 위해 시각화 툴인 RViz를 실행합니다.
```bash
rviz
```
* **[RViz 카메라 세팅]**: 
  1. 화면 좌측 하단 `Add` 버튼 클릭
  2. 상단의 `By topic` 탭 클릭
  3. 트리 메뉴에서 `/camera/rgb/image_raw` 폴더 아래의 `Image` 선택 후 `OK` 누르기
  *(이제 좌측 하단에 로봇 카메라의 실시간 화면이 뜹니다.)*

### [터미널 3] 로봇 수동 조종 (Teleop)
로봇을 키보드로 조종하기 위한 노드를 켭니다.
```bash
rosrun turtlebot3_teleop turtlebot3_teleop_key
```

---

## 3. 실행 및 검증 (테스트)

이제 모든 준비가 끝났습니다. 실시간 QR 인식기 노드를 가동합니다.

### [터미널 4] 실시간 비전 인식기 노드 실행
```bash
cd ~/catkin_ws
source devel/setup.bash
rosrun qr_logistics_robot vision_recognizer_node.py
```

### ✅ 테스트 방법 및 성공 기준
1. **터미널 3(조종 창)**을 선택한 상태에서, 키보드의 `a`(좌회전), `d`(우회전) 키 등을 가볍게 눌러 로봇을 제자리에서 회전시킵니다.
2. 로봇이 회전하다가 **RViz 카메라 창에 QR 박스가 등장하는 순간!**
3. 켜두었던 **터미널 4(비전 인식기)** 창에 번개처럼 아래와 같은 로그가 뜨면 완벽하게 성공입니다!

[INFO] [...]: =====================================================
