# 개발 규칙

## 1. 기본 규칙
- 모든 코드는 한글로 주석을 작성한다.
- 모든 변수명은 영어로 작성한다.
- 모든 함수명은 영어로 작성한다.
- 모든 파일명은 영어로 작성한다.
- 모든 디렉토리명은 영어로 작성한다.
- 모든 주석은 한글로 작성한다.
- 모든 변수명은 영어로 작성한다.
- 모든 함수명은 영어로 작성한다.
- 모든 파일명은 영어로 작성한다.
- 모든 디렉토리명은 영어로 작성한다.

## 2. 미니프로젝트 특별 규칙
- docs 폴더 내부의 파일은 사용자가 요청하지 않는 한 AI가 임의로 수정하지 않는다.
- AI는 @docs/work_spec/ 폴더 내의 작업명세서를 먼저 확인하고, 그에 따라 개발을 진행한다.
- AI를 활용한 개발은 반드시 세세한 작업명세서를 먼저 작성하고, 그에 따라 개발을 진행한다.
- 각 모듈을 개발하기 전에 해당 모듈의 기능과 역할을 정의하는 문서를 작성한다.
    - 해당 모듈이 어떤 기능을 수행하는지
    - 해당 모듈이 어떤 역할을 하는지
    - 해당 모듈이 어떤 입력을 받는지
    - 해당 모듈이 어떤 출력을 반환하는지
    - 해당 모듈이 어떤 예외 상황을 처리하는지
    - 파일명은 [모듈명]_module.md 형식으로 작성한다. (예시: docs/qr_code_module.md)
- 각 모듈을 개발한 후에는 해당 모듈의 기능을 테스트하는 프로그램을 작성한다.
- 각 모듈을 개발한 후에는 해당 모듈의 기능을 문서화한다.
- 프로젝트를 진행한 후에는 당일 진행 상황을 정리하여 [devnote/](devnote/) 내에 md 파일로 작성한다.
    - 파일명은 [날짜]_devnote.md 형식으로 작성한다. (예시: 2026-04-13_devnote.md)
