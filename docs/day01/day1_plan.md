# 1일차 (Day 1) 세부 진행 계획: QR 기반 물류 배송 로봇 시스템

## 1. 개요
* **목표**: ROS/Gazebo 기본 시뮬레이션 환경 구축 및 QR 코드 생성/인식 기능의 모듈화 구현
* **핵심 방향**: 시스템 통합 시 발생할 수 있는 오류를 쉽게 추적하고 수정할 수 있도록, 각 기능을 완전히 독립된 모듈(스크립트 및 ROS 노드)로 분리하여 개발합니다.

---

## 2. 모듈별 상세 계획

### [모듈 1] 기반 환경 구축 (Environment & Infrastructure Module)
* **목표**: 로봇 시스템 구동을 위한 ROS 패키지 및 시뮬레이션 기초 세팅
* **세부 작업**:
  1. ROS Noetic 워크스페이스 (`catkin_ws`) 초기화 및 빌드
  2. 본 프로젝트용 메인 패키지 생성 (예: `qr_logistics_robot`)
  3. Gazebo 시뮬레이션 구동을 위한 기본 런치 파일(`*.launch`) 작성 및 Empty World 구동 테스트
* **독립성/검증**: 다른 코드 없이 `roslaunch` 명령어만으로 시뮬레이터가 정상적으로 켜지는지 확인합니다.

### [모듈 2] QR 코드 생성기 (QR Generator Module)
* **목표**: 물류 데이터 포맷을 정의하고 이를 QR 이미지 파일로 출력
* **세부 작업**:
  1. 배송 로봇이 식별할 데이터 구조 정의 (예: JSON 형식의 `{"item_id": "A123", "destination": [2.5, -1.0], "priority": "high"}`)
  2. Python `qrcode` 라이브러리를 활용하여 텍스트 데이터를 QR 이미지로 변환 및 저장하는 독립 스크립트 작성 (`qr_generator.py`)
* **독립성/검증**: ROS 의존성 없이 단독으로 실행 가능해야 하며, 생성된 이미지 파일을 스마트폰 기본 카메라로 비췄을 때 해당 텍스트/JSON이 정상적으로 보여야 합니다.

### [모듈 3] QR 코드 인식기 (QR Recognizer Module)
* **목표**: 영상/이미지 내의 QR 코드를 인식하고 담겨있는 물류 데이터를 파싱
* **세부 작업**:
  1. OpenCV의 `QRCodeDetector` 또는 `pyzbar` 라이브러리를 활용해 이미지를 읽고 해독하는 핵심 함수 작성
  2. 향후 시뮬레이션 카메라와 연동하기 위해 이 기능을 ROS 노드로 래핑 (`qr_recognizer_node.py`)
  3. 1일차 단독 테스트를 위해 파일 경로를 입력받아 파싱하거나 로컬 웹캠을 켜서 즉시 인식하는 기능 포함
* **독립성/검증**: 모듈 2에서 만든 이미지를 입력으로 주었을 때, 텍스트가 콘솔/터미널 창에 정확하게 파싱되어 출력되는지 확인합니다. 추후 이 모듈에서 파싱된 좌표값만 다른 주행 노드로 넘겨주게 됩니다.

## 3. 1일차 진행 순서 및 체크리스트
- [ ] **Step 1**: ROS 워크스페이스 구축 및 `qr_logistics_robot` 패키지 생성 완료
- [ ] **Step 2**: Gazebo 시뮬레이션 기본 World 띄우기 성공
- [ ] **Step 3**: `qr_generator.py` 작성 및 QR 이미지 파일 생성 테스트 완료
- [ ] **Step 4**: `qr_recognizer_node.py` 작성 및 QR 이미지 데이터 파싱 테스트 완료
- [ ] **Step 5**: (시간 여유 시) 인식된 물류 데이터를 ROS Topic (예: `/target_logistics_info`)으로 퍼블리시(Publish)하는 기능 추가
